# Quick Start Guide — FlowTech Notion Templates

Welcome! You're 5 minutes away from having your new workspace set up in Notion.

---

## What You Downloaded

Your ZIP file contains:

- **Main template file** (`.md`) — The full workspace structure
- **databases/** folder — CSV files with pre-built databases and sample data
- **SETUP-GUIDE.md** — Detailed setup instructions with screenshots

Some templates also include bonus files (SOW templates, review templates, etc.).

---

## Step 1: Import the Template (2 minutes)

1. Open **Notion** in your browser or desktop app
2. In the left sidebar, click **Import** (at the bottom)
3. Select **"Markdown & CSV"**
4. Choose the main `.md` file from your download:
   - AgentOps: `agentops-main.md`
   - Consultant OS: `consultant-os-main.md`
   - PM OS: `pm-os-main.md`
   - SiteOS: `siteos-main.md`
5. Notion creates a new page with all sections, headings, and content

> **Tip:** If any toggle sections don't import correctly, type `/toggle` in Notion to recreate them manually.

---

## Step 2: Import the Databases (3 minutes)

For each CSV file in the `databases/` folder:

1. Inside your new template page, scroll to the relevant section
2. Type `/database` and select **"Database - Inline"**
3. Name the database (e.g., "Projects", "Tasks", "Clients")
4. Click the **"..."** menu at the top-right of the database
5. Select **"Merge with CSV"**
6. Choose the matching CSV file
7. Notion creates all columns and imports the sample data

Repeat for each CSV. Your SETUP-GUIDE.md lists the exact order and column types to set.

---

## Step 3: Set Up Relations (5 minutes)

Relations connect your databases so data flows between them (e.g., tasks link to projects, invoices link to clients).

1. Open the **SETUP-GUIDE.md** included in your download
2. Follow the "Set Up Relations" section — it lists exactly which databases to link
3. For each relation: add a "Relation" property, point it to the target database

> **Why this matters:** Relations are what make this template powerful. A single project page will show all its tasks, risks, invoices, and notes — no clicking around.

---

## Step 4: Make It Yours

1. **Delete the sample data** — it's there to show the expected format, not for you to keep
2. **Enter your real data** — start with the most important database (usually Projects or Clients)
3. **Customise select options** — add statuses, categories, and tags that match your workflow
4. **Set up dashboard views** — the SETUP-GUIDE walks you through creating filtered views
